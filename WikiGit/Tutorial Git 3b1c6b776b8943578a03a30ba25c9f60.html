Tutorial Git

Git pode parecer complicado mas sua base é bem simples. Existem alguns comandos essenciais que você precisa saber para conseguir realizar suas atividades diárias na parte de programação, aqui eu registrarei os principais comandos para facilitar a vida de qualquer Dev. necessitado.

Para o que serve o Git ?

Git é um sistema de controle de versão. Ele permite que desenvolvedores gerenciem e acompanhem as alterações no código fonte ao longo do tempo. O Git facilita a colaboração, pois permite que várias pessoas trabalhem no mesmo projeto sem se atrapalharem.

Para compartilhar o projeto com outras maquinas você precisa subir seu projeto para um repositório remoto, ou seja em algum lugar fora do seu computador (como se fosse na nuvem), como o Github

Do que eu preciso para usar o Git (e o GitHub) ?

Você precisará de um computador com Git instalado, uma conta no GitHub e muita determinação. A instalação do Git varia dependendo do sistema operacional que você está usando, mas geralmente é um processo simples. Quanto ao GitHub, é grátis criar uma conta e começar a usar. Vou deixar os passos abaixo, mas se você for tech pode pular toda essa parte.

Instalação Git - Windows

A instalação para Windows é bem simples e não precisa da senha do administrador. Aqui estão os passos a serem seguidos :

  1. Faça o download do Git a partir do site oficial (https://git-scm.com/download/win)
  1. Execute o arquivo de instalação e siga as instruções na tela. Não há necessidade de alterar as opções padrão.
  1. Para verificar se a instalação foi bem-sucedida, abra o prompt de comando e digite: git --version. Deverá aparecer a versão do Git instalada.

Caso haja alguma duvida eu deixei um passo a passo bem ilustrativo abaixo:

Iniciação ao GitHub

Para começar a usar o GitHub, primeiro você precisa criar uma conta. Acesse https://github.com e clique em "Sign up". Preencha as informações necessárias e siga as instruções na tela. Uma vez que sua conta esteja criada, você pode começar a criar seus próprios repositórios e colaborar com outros projetos.

Eu imagino que você já tenha conta, mas se não eu acredito no seu potencial.

Começando um repositório

Para começar um repositório Git, primeiro você precisa criar um novo diretório no seu computador. Você pode fazer isso navegando até o local onde você deseja que o novo repositório esteja. Por exemplo, se você quiser criar um novo repositório na sua pasta Documentos, primeiro você navega até a pasta Documentos.

Uma vez que você esteja no diretório desejado, você pode criar um novo diretório para o seu repositório. Clique com o botão direito do mouse em um espaço vazio dentro do diretório e selecione a opção "Novo > Pasta" do menu que aparece. Digite o nome que você deseja para o seu repositório e pressione Enter.

Agora que você tem um novo diretório, você pode inicializá-lo como um repositório Git. Para fazer isso, clique com o botão direito do mouse na pasta que você acabou de criar e selecione "Git Bash Here" do menu que aparece. Isso abrirá um terminal Git Bash.

No terminal, digite o comando git init. Este comando inicializa um novo repositório Git no diretório atual. Você deve ver uma mensagem dizendo "Initialized empty Git repository in [seu diretório]". Isso significa que você agora tem um repositório Git vazio pronto para você começar a adicionar e controlar arquivos.

Pronto! Agora você tem um repositório Git vazio, pronto para você começar a usar. A partir daqui, você pode começar a adicionar arquivos ao seu repositório, fazer commits de suas alterações e muito mais.

A partir daqui temos dois caminhos principais :

  • Criar um repositório local e subir para a nuvem
  • Clonar um repositório remoto existente para a sua maquina

*Para qualquer uma das situações o git init é necessário, ou seja não esqueça dele!!!

IMPORTANTE!!!!

Por padrão a branch principal do Git vem nomeada como master. Porem a comunidade tecnológica entrou em consenso que esse termo é racista e não deve ser utilizado. Todos os comandos a seguir assumem que o nome da sua branch já foi alterada, mas caso ainda esteja como master utilize esse comando

  • Comando para mudar o nome da branch
git branch -M main

O próprio GitHub oferece opções de código para você iniciar seu repositório, mas caso haja duvida olha algum dos tutoriais abaixo:

Criar um repositório local e subir para a nuvem

Com sua pasta já criada e o Git iniciado você apenas a alguns passos de enviar seu repositório local para a nuvem.

Primeiro você precisa criar o repositório remoto que guardará seus projetos no Github.

Isso pode ser feito rapidamente através do botão “new” localizado na home:

Você terá que colocar preencher algumas informações como nome, privacidade e descrição (opcional), depois apenas clique em “create repository”.

Agora na linha de comando (Git Bash, Powershell ou qualquer um que você estiver usando) você vai precisar digitar o seguinte comando:

  • Criando conexão com o repositório remoto
 git remote add origin (linkDoGitHub)

Agora você precisa subir os arquivos que estão na sua maquina, os comandos é :

git add .

irá adicionar todos os arquivos do diretório para o Git.

Em seguida, você precisa fazer um commit, que é uma espécie de "salvamento" do seu projeto até aquele ponto.

git commit -m "mensagem legal"

Por fim, você precisa 'empurrar' essas alterações para o GitHub com o comando:

git push origin main

Prontinho! Agora seu repositório já está no ar.

Clonar um repositório remoto existente para a sua maquina

Para usar um repositório remoto já existente você precisa do comando “git clone”

siga os passos abaixo:

  • Copiando link do repositório remoto

  • Comando para clonar o repositório
git clone (linkDoGitHub)
